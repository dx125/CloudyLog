# Puff — Deferred Work

Features and infrastructure from the [handoff](puff-handoff.md) and [design book](puff-design-book.html) that are **intentionally not in the current build** (which covers Phase 0 — pilot hardening and Phase 1 — launch). Items are grouped by roadmap phase; the last section lists infrastructure deferred for practical reasons (accounts, store setup) rather than roadmap order.

## Phase 2 — Retention

- **Trigger food detective** — meal logging with quick tags; correlate food to gas 4–12 h later (FODMAP-elimination territory). The data model is already ready: meals are a second `type` on the same append-only `events` table, not a new system. This is the flagship Pro feature; once it ships, the paywall copy should lead with it (handoff §4).
- **Doctor report** — clean PDF export of frequency history; generated by a Supabase Edge Function.
- **Duels** — head-to-head weeks. Needs `duels` + `duel_members` tables with RLS (members read shared duel aggregates only), Supabase Realtime channels for live scores, Edge-Function validation of results, and simple anti-cheat (physical upper bound on plausible taps/hour). Free tier joins one duel; Pro unlimited + create. **Blocker before any social ships: profanity filter and report flow for duel/league names** (handoff §9). The Duels tab currently shows a "coming soon" teaser.
- **Home-screen widget** — one-tap logging via `home_widget` with an app-group-shared count.
- **Watch app** — native SwiftUI companion over WatchConnectivity writing into the local event store through a platform channel (Flutter doesn't run on watchOS). Wear OS follows the same pattern later.
- **Cosmetics store** — sound packs ($0.99–1.99), themes, Gust seasonal skins; giftable IAPs outside the subscription. Free tier keeps the single default "pop" sound (currently a system click placeholder — a real recorded pop asset is part of this work).
- **Custom quick tags beyond the built-in set** — Pro already gets the full built-in set + user-defined tags in this build; richer tag management (reorder, colors, per-tag stats) lands with the food detective.
- **Full Wrapped** — monthly editions and server-rendered cards (Edge Function → PNG behind a CDN so cards look identical everywhere). Current build ships the basic client-rendered year card. Wrapped-at-scale prep: pre-compute datasets during the last week of the period; plan for ~20× normal traffic on Wrapped day.

## Phase 3 — Expansion

- **Leagues** — household/group leaderboards on top of duels.
- **Health app integration** — sleep, cycle, exercise overlays on Puff charts (HealthKit / Health Connect).
- **Localization** — re-add non-English locales. The ARB/l10n infrastructure is kept in the app; the previous Spanish file was removed with the CloudyLog copy. Voice rules (one joke per screen, sentence case) need per-locale review, not literal translation.
- **Gut-comfort tracker expansion** — only if the data supports broadening beyond the counter under the same brand.

## Deferred infrastructure (not phase-gated)

- **RevenueCat** — `purchases_flutter` SDK + RevenueCat webhook → Edge Function mirroring entitlements into the `entitlements` table. The current build hand-rolls nothing: purchases go through a `PurchaseGateway` seam whose dev implementation calls the `activate_mock_pro()` RPC; RevenueCat replaces that one class. Pricing: $2.49/mo, $17.99/yr, $29.99 lifetime — configure as store products.
- **Apple / Google sign-in** — anonymous sessions currently upgrade via email+password only (`auth.updateUser`). Add `signInWithApple` / Google once bundle IDs and OAuth clients exist.
- **Sentry + PostHog** — crash reporting and the product funnel (first tap → D7 retention → paywall view → conversion) should be instrumented from the first shipped build. PostHog feature flags double as remote config for paywall copy and price testing ($1.99–2.99 is worth real money).
- **Push notifications** — FCM/APNs; permission is requested only after the first streak, never on first launch.
- **Hourly global stats** — the `pg_cron` job currently rolls up daily (03:10 UTC); move to hourly when volume justifies ("world average · updated hourly" caption becomes literal).
- **Close direct PostgREST exposure** — the app only calls Edge Functions, but the underlying REST endpoints still exist and are guarded by RLS (a crafted client could hit them and see exactly what RLS allows — nothing more). To hard-close them, stop exposing `public` over PostgREST and switch the functions to a direct DB connection or a dedicated exposed schema. Do this alongside the RevenueCat work when the API surface is next touched.
- **Events volume scaling** — partition `events` by month early; per-user rollup tables (`daily_user_stats`) maintained by trigger/cron so charts never scan raw events; read replicas when dashboards get heavy.
- **Rewarded-ad ceiling** — only if free-tier monetization is ever needed, a single rewarded ad ("watch to unlock today's detailed breakdown"); wait for data.
- **Store metadata / ASO** — title "Puff — Fart Counter & Gut Tracker", mascot-led screenshots, PG language ("toot" everywhere user-facing).
